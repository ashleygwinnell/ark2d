00015  /* png_info is a structure that holds the information in a PNG file so
00016  * that the application can find out the characteristics of the image.
00017  * If you are reading the file, this structure will tell you what is
00018  * in the PNG file.  If you are writing the file, fill in the information
00019  * you want to put into the PNG file, using png_set_*() functions, then
00020  * call png_write_info().
00021  *
00022  * The names chosen should be very close to the PNG specification, so
00023  * consult that document for information about the meaning of each field.
00024  *
00025  * With libpng < 0.95, it was only possible to directly set and read the
00026  * the values in the png_info_struct, which meant that the contents and
00027  * order of the values had to remain fixed.  With libpng 0.95 and later,
00028  * however, there are now functions that abstract the contents of
00029  * png_info_struct from the application, so this makes it easier to use
00030  * libpng with dynamic libraries, and even makes it possible to use
00031  * libraries that don't have all of the libpng ancillary chunk-handing
00032  * functionality.  In libpng-1.5.0 this was moved into a separate private
00033  * file that is not visible to applications.
00034  *
00035  * The following members may have allocated storage attached that should be
00036  * cleaned up before the structure is discarded: palette, trans, text,
00037  * pcal_purpose, pcal_units, pcal_params, hist, iccp_name, iccp_profile,
00038  * splt_palettes, scal_unit, row_pointers, and unknowns.   By default, these
00039  * are automatically freed when the info structure is deallocated, if they were
00040  * allocated internally by libpng.  This behavior can be changed by means
00041  * of the png_data_freer() function.
00042  *
00043  * More allocation details: all the chunk-reading functions that
00044  * change these members go through the corresponding png_set_*
00045  * functions.  A function to clear these members is available: see
00046  * png_free_data().  The png_set_* functions do not depend on being
00047  * able to point info structure members to any of the storage they are
00048  * passed (they make their own copies), EXCEPT that the png_set_text
00049  * functions use the same storage passed to them in the text_ptr or
00050  * itxt_ptr structure argument, and the png_set_rows and png_set_unknowns
00051  * functions do not make their own copies.
00052  */
